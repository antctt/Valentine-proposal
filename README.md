## Deployment to Vercel

This project is configured for easy deployment to Vercel:

1. **Via Vercel CLI:**
   ```bash
   # Install Vercel CLI globally
   npm i -g vercel
   
   # Deploy to Vercel
   vercel
   ```

2. **Via Vercel Dashboard:**
   - Go to [vercel.com](https://vercel.com)
   - Click "New Project"
   - Import this repository
   - Click "Deploy" (no build configuration needed)

3. **Via GitHub Integration:**
   - Connect your GitHub account to Vercel
   - Import this repository
   - Vercel will automatically deploy on every push

4. **Via GitHub Actions (Automatic Deployment):**
   - A GitHub Actions workflow is set up for automatic deployment
   - Required GitHub Secrets (add these in repository settings):
     - `VERCEL_TOKEN`: Your Vercel access token (from Vercel Account Settings â†’ Tokens)
     - `VERCEL_ORG_ID`: Your Vercel organization ID (from project settings)
     - `VERCEL_PROJECT_ID`: Your Vercel project ID (from project settings)
   - The workflow automatically deploys:
     - Production deployments on push to `main` branch
     - Preview deployments for pull requests

The `vercel.json` configuration file is already set up for static site deployment.
